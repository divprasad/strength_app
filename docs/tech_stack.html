Core Technologies

The tooling choices emphasizing speed, local reliability, and visual coherence.

01 Next.js (App Router)

Why we chose it: The foundation of the app. It unifies the frontend interfaces cleanly with the /api routing necessary to connect the local interface to the SQL backend database. Even though most data rendering occurs client-side for offline speed, Next.js gives us the infrastructure to seamlessly convert into a Progressive Web App (PWA).

02 Dexie.js (IndexedDB)

Why we chose it: Under the hood, this replaces traditional React State arrays (like Redux or Zustand). Dexie acts as a minimalist wrapper over the browser's native IndexedDB. By treating the phone's browser as the primary database, the application becomes completely disconnected from network dependency, providing the crucial "0ms ping" required for a gym tracker app.

03 Prisma & SQLite

Why we chose it: This is the server-side source of truth. Prisma forces strict relational typing, ensuring our exercise UUIDs map flawlessly to the logged sets. SQLite was selected over heavier dialects like PostgreSQL specifically because this is designed as a single-player application; SQLite runs literally anywhere (Raspberry Pi, NAS, or basic laptop) without requiring complex container setups.

04 Tailwind CSS & Shadcn/UI

Why we chose it: For visual excellence and development speed. Shadcn provides unstyled accessible generic components that we bend via Tailwind. It achieves aesthetic density without polluting the javascript payload.

05 Docker & PWA Architecture

Why we chose it: Bypassing the native iOS/Android App Stores while retaining a full-screen, native-feeling app experience. The public/sw.js caches all application shells to allow the app to launch instantly while completely offline. The docker-compose.yml file orchestrates the local next start process while enforcing strict SQLite persistence through named volumes (db-data), allowing this app to run infinitely safely on a cheap local NAS or Raspberry Pi.
