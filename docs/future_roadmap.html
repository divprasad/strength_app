Road to Production

The explicit steps needed to finalize this app for a fast, single-user mobile experience.

Phase 1 PWA Transformation [COMPLETED]

To achieve a "snappy and responsive" app natively over Android, we bypass the app store entirely by completing a Progressive Web App implementation.

  • Manifest File: Created a `manifest.json` injected via Next.js to provide standard launcher icon parameters and standalone display configurations.
  • Service Workers: Cached the static `.js` routing bundles locally via `sw.js` (cache-first static, network-first SSR).

Phase 2 Local-Network Backend Hosting [COMPLETED]

Because there is no authentication mechanism needed yet, the endpoint lives fully locally on the host network.

  • Dockerization: Containerized the `next start` command along with a named persistent volume mapping for the `prisma/dev.db` file (`db-data`).

Phase 3 Multi-User & Offline PIN Auth

Transitioning the app from a strictly single-player experience to a family/shared environment where multiple users can hit the same local server from different devices.

  • Context Switching: Implementing user switching that partitions the local IndexedDB state by `userId`.
  • Local Auth: Creating an offline-capable 4-digit PIN lock screen so workouts stay private, validating against salted IndexedDB hashes rather than round-tripping to cloud OAuth.

Phase 4 Production Clean-ups

With the PWA and Docker infrastructure shipped, the remaining work focuses on reliability, visibility, and polish for daily use.

  • TypeScript Bundle Sweeping: Reviewing existing component logic and converting completely dynamic Next.js API routes to `force-static` wherever feasible to minimize rendering cycles.
  • Robust Logging: Upgrading the silent database backup system built into `api/workouts/route.ts` to surface user-facing confirmation modals.
