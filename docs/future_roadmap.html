Phase 1 PWA Transformation

To achieve a "snappy and responsive" app natively over Android, we shouldn't build an `.apk`. We bypass the app store entirely by completing a Progressive Web App implementation.

  • Manifest File: Creating a `manifest.json` injected via Next.js to provide standard launcher icon parameters, standalone display configurations (hiding the URL bar), and custom theme colors.
  • Service Workers: Caching the static `.js` routing bundles locally. With Dexie managing the data and Service Workers managing the frontend UI, the application will boot instantly from the Android desktop without requiring a network handshake.

Phase 2 Local-Network Backend Hosting

Because there is no authentication mechanism needed, hosting this on public Vercel/Render introduces security risks. The endpoint should live fully locally on the host network.

  • Dockerization: Containerizing the Next.js `npm run start` command along with the volume mapping for the `prisma/dev.db` file allows this app to persist indefinitely on a Raspberry Pi, Synology NAS, or any home server stack via port 3000.
  • Tailscale / VPN Proxy: Accessing the internal IP natively from an Android device while actively at the gym, leveraging Tailscale or a similar mesh proxy, completely avoiding domain purchase or external ingress management.

Phase 3 Production Clean-ups

  • TypeScript Bundle Sweeping: Reviewing existing component logic and converting completely dynamic Next.js API routes to `force-static` wherever feasible to minimize rendering cycles constraint.
  • Asset Minification: Compressing existing UI constants into the smallest possible `Next/Image` and raw UI footprint.
  • Robust Logging: Upgrading the silent database backup system built into `api/workouts/route.ts` to surface user-facing confirmation modals if syncing to the home server repeatedly timeouts.
