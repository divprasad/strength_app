✓ Phase 1 PWA Transformation Shipped Mar 29, 2026

The application is a fully installable Progressive Web App via Serwist, a modern service worker toolkit built on Google's Workbox.

  • Manifest File: A manifest.ts injected via Next.js provides standard launcher icon parameters, standalone display configuration (hiding the URL bar), and custom theme colors (#131519).
  • Service Workers: Serwist precaches all static bundles and applies runtime caching strategies. Combined with Dexie managing the data layer, the application boots instantly from Android/iOS desktops without requiring a network handshake.
  • App Icons: SVG source icons with a generated PNG set at 192px and 512px (including maskable variants and Apple Touch icons) stored in public/icons/.

✓ Phase 2 Local-Network Backend Hosting Shipped Mar 29, 2026

The application is fully Dockerized for self-hosted deployment on any local network device.

  • Dockerization: A multi-stage Dockerfile (deps → builder → runner) on Node 20 Alpine produces a lean ~180 MB image. A docker-compose.yml manages the single service with a named volume (db-data) persisting the SQLite database across container rebuilds.
  • Entrypoint Automation: The docker-entrypoint.sh script handles fresh migration syncing, prisma migrate deploy, conditional seeding on first boot, and then starts the Next.js standalone server.
  • Tailscale / VPN Proxy: The containerized app runs on port 3000, accessible from any device on the local network. For gym access outside the home, a Tailscale mesh VPN provides secure connectivity without domain purchases or public ingress.

◆ Phase 3 Production Hardening In Progress

With the PWA and Docker infrastructure shipped, the remaining work focuses on reliability, visibility, and polish for daily use.

  • Sync Status UI: Adding a user-facing sync indicator in the app shell (green/amber/red) and surfacing toast notifications when background pushes to the server fail consecutively — eliminating silent data-loss anxiety.
  • End-to-End Docker Validation: Running through a full smoke test cycle on the target hardware: container boot → workout logging → sync push → backup verification → container restart → data persistence confirmation.
  • TypeScript Bundle Sweeping: Reviewing component logic and converting dynamic API routes to force-static wherever feasible to minimize rendering cycles.
  • Test Coverage Expansion: Adding targeted E2E and unit tests for the newly tabbed Exercises UI, the command palette actions, and the PWA install flow.
  • Asset Minification: Compressing UI constants and images to the smallest possible footprint for mobile load times.
