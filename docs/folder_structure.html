The Directory Tree

antigravity_strength/
├── docker-compose.yml    # Orchestrates local hosting and storage
├── Dockerfile            # Builds the Next.js production image
├── docs/                 # Documentation (You are reading this!)
├── prisma/               # Database definitions and active SQLite files
│   ├── dev.db            # The local database
│   ├── schema.prisma     # The SQL definitions mapped to TypeScript
│   └── migrations/       # SQL history
├── public/               # Static assets (images, icons, manifest config)
│   ├── manifest.json     # PWA Installer Configuration
│   └── sw.js             # Service Worker for offline caching
└── src/
    ├── app/              # Next.js App Router (Pages & Background API)
    ├── components/       # Interface layer (React components)
    ├── hooks/            # Custom logic loops specific to React
    ├── lib/              # The "Brain" (Database config, Sync Engines, Utils)
    └── types/            # TypeScript Domain Definitions

/src/app (The Router)

This defines what the user sees when they type a URL. For example, /src/app/workouts/page.tsx dictates the view at localhost:3000/workouts. It also holds the critical /api directory, which serves as the ingress node for the synchronization engine to write to Prisma.

/src/components (The Interface)

Built primarily around specialized domain sections (e.g., /components/workout/, /components/analytics/) utilizing specialized UI primitive components like shadcn/ui. This folder holds zero networking logic, purely interfacing with src/lib methods.

/src/lib (The Core Logic)

The most important folder in the project. It handles all state management and heavy lifting:

  • db.ts: Initializes the local Dexie browser cache and the offline synchronization queue.
  • repository.ts: The unified controller. React components fire requests here, and it handles writing to IndexedDB and enqueuing background tasks simultaneously.
  • syncEngine.ts: The background worker that flushes stored browser logic safely into the SQLite files when network requests clear.
